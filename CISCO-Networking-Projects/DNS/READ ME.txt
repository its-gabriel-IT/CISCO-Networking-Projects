Cisco DNS Set Up 
-----------------------------------------------------------------------------------------------------------------------------

Overview

This project demonstrates how to configure a Domain Name System (DNS) server to resolve domain names to IPv4 addresses using A records, alongside DHCP for automatic IP assignment and static routing for end-to-end connectivity. The lab simulates a small enterprise network where devices can communicate using domain names instead of IP addresses.
-------------------------------------------------------------------------------------------------------------------------------

Objectives

*Configure a DNS server to resolve domain names to IP addresses
*Implement DHCP for dynamic IP address allocation
*Set up static routing between network segments
*Verify domain resolution and end-to-end connectivity
*Practice real-world network troubleshooting techniques
--------------------------------------------------------------------------------------------------------------------------------

Technologies & Concepts Used

* DHCP (Dynamic Host Configuration Protocol)
*DNS (Domain Name System)
*Cisco Router Configuration via CLI
*Interface Configuration (GigabitEthernet, Serial)
*Basic Layer 2 Switching
*ICMP Testing (ping, connectivity validation)
--------------------------------------------------------------------------------------------------------------------------------

Network Architecture

Topology Summary:
* 1 Routers
* 1 Managed Switches (LAN access layer)
* Multiple end devices (PCs, Laptops & Server)
* IEE 802.3 UTP Ethernet Cable

--------------------------------------------------------------------------------------------------------------------------------

Skills Demonstrated

* DNS configuration and name resolution
*DHCP setup and IP management
*Router interface configuration using Cisco CLI
*Static routing implementation
*Layered troubleshooting methodology (OSI model approach)
